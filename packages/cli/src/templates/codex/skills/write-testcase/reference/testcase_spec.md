# 测试用例设计规范

## 核心原则

**从用户视角设计测试用例，禁止技术术语**

用户视角关注：
- 用户能看到什么（界面、提示、结果）
- 用户能操作什么（按钮、输入框、菜单）
- 用户能感知到什么（功能是否正常、响应是否及时、错误提示是否清晰）

## 用例设计模块

- 配置测试
- 功能测试
- 组合测试
- 协议一致性测试
- 异常测试
- 性能测试
- 兼容性测试
- 可维护性测试

## 设计方法

### 场景法（优先使用）

基于用户使用场景设计，每个场景包含：
1. 场景名称
2. 用户角色（管理员、操作员等）
3. 使用环境（Web界面、EZWall客户端、VM平台）
4. 使用目的
5. 前置条件（从用户角度）
6. 操作步骤（界面操作）
7. 预期结果（用户感知）

**示例**：
```
前置条件：管理员已登录VM平台，已创建电视墙"TW001"
测试步骤：
1. 在电视墙列表点击"TW001"的"配置"按钮
2. 勾选"叠加用户名"、"叠加IP地址"
3. 输入自定义内容"机密文档"
4. 点击"保存"
预期结果：
1. 页面显示"保存成功"
2. 电视墙显示水印："admin 192.168.1.100 机密文档"
```

### 等价类划分

划分有效和无效输入，从用户角度设计

### 边界值分析

测试边界值及边界外的值

### 错误推测

推测用户可能遇到的错误场景（非法输入、误操作等）

## 测试场景覆盖

- 正面场景：验证基本功能
- 负面场景：验证无效输入、错误操作
- 异常场景：重启、断网、并发操作等
- 边界条件：最大值、最小值、特殊字符
- 数据完整性：不同终端配置一致性

## 检查清单

### 前置条件
- [ ] 从用户角度描述（用户已登录 ✅ / 数据库连接正常 ❌）
- [ ] 包含用户操作环境和权限

### 测试步骤
- [ ] 描述界面操作（点击、输入、选择）
- [ ] 避免技术术语（无模块名、函数名、数据库表名）

### 预期结果
- [ ] 描述用户看到的结果（提示信息、显示效果）
- [ ] 避免技术实现描述（无返回码、无数据库字段）

### 整体
- [ ] 非技术人员可理解
- [ ] 可手动执行
